Deaccessioning - User Notes

  • To deaccession a plant, go to the accession page for that plant
    1. Change all quantities to 0 (zero)
    2. Change all locations to 0 (zero)
    3. Change Currently: to:
      • deceased - if plants are dead
      • culled - if plants removed for reasons other than death
    4. Change Assigned to: to DELETE## where ## is year of deaccession
      • ie plants deleted in 2020 should be placed in DELETE20
      • This allows plants to be (re)accessioned should it be determined they were removed in error
        Reaccessioned plants will retain all historical notations
      • Also allows previously researched data to be retained for future use or copying to new accessions of same species
      • Drop-down list is auto populated from existing codes
        • Each year, a new code will need to be manually entered for first deaccession via commandline as follows:
        • Assumes first deaccession for 2021 and first plant to be deaccessioned is 198912345, change those values as required
        • From the MySQL command line (local or remote)
        • update gh_inv set projnum='DELETE21' where codeno = 198912345;
